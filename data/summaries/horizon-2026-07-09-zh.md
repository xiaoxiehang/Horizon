## [Bun 将高性能 JavaScript 运行时从 Zig 重写为 Rust](https://bun.com/blog/bun-in-rust) ⭐️ 8.0/10

Bun 团队已正式将其高性能 JavaScript 运行时从 Zig 重写为 Rust，实现了二进制大小缩减 20%、性能提升 5%以及稳定性的增强。 这一重大架构转变凸显了在系统编程中，Rust 严格的类型系统和内存安全性日益受到青睐，同时也引发了对 Zig 在大型项目中长期可行性的质疑。它还展示了在强大测试套件的配合下，大语言模型（LLM）辅助大规模代码库迁移的新兴能力。 此次重写在很大程度上借助了大语言模型，并利用 Bun 广泛的测试套件来获取可验证的反馈，特别指出使用了 Fable 等模型而非仅仅是类似 Opus 的模型。该转换还解决了内存泄漏问题，并通过结合 Rust 重写、ICU 更改和相同代码折叠技术将二进制大小缩减了 20%。

hackernews · afturner · 7月8日 21:49 · [社区讨论](https://news.ycombinator.com/item?id=48837877)

**背景**: Bun 是一个受欢迎的高性能一体化 JavaScript 运行时和工具包，专为速度而设计，最初使用 Zig 编程语言构建。Zig 以其简单性和无隐藏控制流而闻名，但批评者经常指出，与 Rust 相比，它过于冗长且缺乏泛型等高级特性。Rust 通过其严格的所有权模型和类型系统，在没有垃圾回收器的情况下保证了内存安全。

**社区讨论**: 社区指出，重写在修复内存泄漏和缩小二进制大小方面的成功，反映了 Zig 的冗长和缺乏现代抽象的缺点。关于大语言模型在促进此类大规模重写中的作用也存在激烈辩论，用户强调强大的测试套件使 LLM 辅助移植非常有效，尽管一些人对迁移的隐藏成本仍持怀疑态度。
