## [SpaceXAI 开源 Grok Build AI 编程智能体](https://www.ithome.com/0/977/265.htm) ⭐️ 9.0/10

SpaceXAI 正式开源了 Grok Build，这是一个集成在终端中的 AI 编程智能体，能够处理从规划到 Git 提交的全流程编码工作。包括智能体逻辑、工具和用户界面组件在内的完整源代码现已在 GitHub 上公开，供公众使用和修改。 这一举措显著提高了透明度，允许开发者检查 AI 如何构建上下文和调度工具调用，从而促进信任和定制化。它还支持本地优先运行，使用户能够使用自己的本地推理模型运行该智能体，同时解决了近期关于用户代码上传的隐私担忧。 开源版本包括用于构建上下文的智能体链路、用于代码编辑和命令执行的工具、带有内联差异查看功能的终端 UI，以及支持 MCP 服务器和插件的扩展系统。用户可以自行编译该工具，并通过 config.toml 文件进行配置，以指向其首选的本地或远程推理端点。

## [X 平台将零保留开源全部代码并引入第三方核验](https://www.ithome.com/0/977/233.htm) ⭐️ 9.0/10

埃隆·马斯克宣布，X 平台将在完成内部安全审查后，零保留地开源其全部代码库。该平台还将引入第三方审核机构，确保对外公开的代码与线上实际运行的代码完全一致。 此举通过解决公众对“形式化透明”的质疑，挑战了行业惯例，并可能显著影响 Lens Protocol 等去中心化社交媒体竞争对手。其目标是在保留中心化平台网络效应的同时，获得完全技术透明带来的信任优势。 与 2025 年仅公开推荐算法等部分代码不同，此次举措涵盖所有系统模块和功能，毫无例外。然而，全面暴露架构细节带来重大安全风险，因此发布前的内部安全审查对于防止潜在漏洞被利用至关重要。

rss · IT之家 · 7月15日 13:53

## [Thinking Machines Lab 发布 9750 亿参数开源多模态模型 Inkling](https://www.wired.com/story/thinking-machines-lab-releases-its-first-model-inkling/) ⭐️ 9.0/10

Thinking Machines Lab 发布了 Inkling，这是一个拥有 9750 亿参数的开源权重多模态模型，能够在 Apache 2.0 许可下处理文本、图像和音频。此次发布还包括一个名为 Inkling-Small 的 2760 亿参数较小版本，并引入了可控制思考努力程度的机制以平衡成本和性能。 此次发布为企业提供了一个强大的、可定制的闭源模型替代方案，用于代理 AI 工作负载，允许本地部署和微调。它通过提供优先考虑事实输出而非审查的高性能开放权重来挑战既定巨头，可能使竞争格局向开放生态系统倾斜。 Inkling 采用混合专家（MoE）架构，拥有 410 亿活跃参数，在 SWE-bench Verified 上得分 77.6%，优于 Nvidia Nemotron 3，但在多个编码基准测试中落后于中国的 GLM 5.2 和 DeepSeek V4 Pro。该模型旨在直接回答争议性话题，并可通过 Hugging Face 和公司的 Tinker API 获取。

rss · Wired · 7月15日 18:05

**背景**: 代理 AI 指的是能够自主规划、做出决策并使用数字工具执行多步任务的系统，而不仅仅是对单个提示做出响应。开放权重模型允许开发人员访问和微调模型的参数以进行定制，这与训练过程也完全透明的完全开源模型不同。对于希望在利用最先进能力的同时控制其 AI 基础设施的企业来说，这种区别至关重要。
